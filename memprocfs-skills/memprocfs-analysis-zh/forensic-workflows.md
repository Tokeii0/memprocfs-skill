# MemProcFS 取证工作流

本文档概述了使用 MemProcFS 执行常见取证任务的结构化工作流。这些工作流旨在实现全面性和可重复性。

## 工作流一：初始系统分类

**目标**：快速收集有关内存获取时系统状态的基本信息。

1.  **系统信息**：检查 `sys/sysinfo` 目录以识别操作系统版本、体系结构和其他基本详细信息。
2.  **进程列表**：通过检查 `proc` 目录的内容列出所有正在运行的进程。注意可疑或意外的进程名称。
3.  **网络连接**：通过检查 `sys/net` 目录分析活动网络连接。寻找不寻常的远程地址或端口。
4.  **用户活动**：通过浏览 `sys/users` 目录识别已登录用户及其活动。

## 工作流二：恶意软件检测与分析

**目标**：识别和分析系统上的潜在恶意软件。

1.  **运行 FindEvil**：使用 `forensic/findevil` 模块自动扫描常见的恶意软件迹象，例如代码注入、挂钩函数和隐藏进程。
2.  **YARA 扫描**：使用带有全面规则集的 `forensic/yara` 模块对所有进程内存执行 YARA 扫描。
    ```bash
    # 在 MemProcFS 文件系统内的示例命令
    cat /forensic/yara/all_processes > yara_results.txt
    ```
3.  **分析可疑进程**：对任何标记为可疑的进程执行更深入的分析：
    -   检查其内存映射 (`proc/<pid>/memmap`)。
    -   转储其可执行文件和模块以进行静态分析 (`proc/<pid>/bin`)。
    -   检查其网络连接和打开的句柄。

## 工作流三：文件和数据恢复

**目标**：从内存中恢复文件、数据和工件。

1.  **NTFS 分析**：浏览 `forensic/ntfs` 模块以分析 NTFS 文件系统元数据并可能恢复已删除的文件。
2.  **预取文件**：检查 `forensic/prefetch` 目录以识别最近执行的应用程序。
3.  **时间线生成**：使用 `forensic/timeline` 模块创建系统活动的时间线。这有助于关联事件并建立操作顺序。
4.  **Web 工件**：使用 `forensic/web` 模块恢复浏览历史、缓存和其他与网络相关的工件。
